# VolumeController - Windows 后台应用自动静音工具

## 概述

VolumeController 是一款 Windows 系统音量控制工具，能够自动检测并管理后台运行进程的音量。当应用程序切换到后台时自动静音，切换回前台时自动恢复音量，同时支持白名单机制豁免指定应用。

## 功能特性

- **自动静音后台应用**：实时监测前台窗口，自动将非前台应用的音频静音
- **自动恢复前台音量**：应用切换回前台时立即取消静音，恢复原始音量状态
- **白名单机制**：指定应用不受自动静音影响，始终保持正常音量
- **系统托盘运行**：最小化到系统托盘，不占用任务栏空间
- **图形化设置界面**：提供完整的设置窗口，方便管理白名单和参数
- **开机自启动**：支持通过注册表实现开机自动运行
- **配置导入/导出**：JSON 格式配置文件，方便备份和迁移
- **日志记录**：完整的操作日志，支持日志轮转，便于排查问题

## 使用方法

### 启动程序

```bash
python main.py
```

程序启动后会自动最小化到系统托盘区域。

### 系统托盘操作

| 操作 | 功能 |
|------|------|
| 左键单击托盘图标 | 切换启用/禁用状态 |
| 右键单击托盘图标 | 打开菜单 |

右键菜单选项：

- **启用/禁用自动静音**：切换功能开关
- **设置**：打开设置界面
- **退出**：退出程序（自动恢复所有被静音的应用音量）

### 托盘图标状态

- 🟢 **绿色图标**：自动静音功能已启用
- 🔴 **红色图标**：自动静音功能已禁用

## 设置界面

通过托盘菜单的"设置"选项打开设置界面，包含三个标签页：

### 常规设置

| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| 启用自动静音功能 | 开启/关闭自动静音 | 开启 |
| 开机自动启动 | 通过注册表实现开机自启 | 关闭 |
| 检测间隔 | 前后台状态检测频率（毫秒） | 500 |

**检测间隔建议**：
- 300-500ms：响应迅速，适合频繁切换窗口的场景
- 500-1000ms：平衡性能与响应，推荐值
- 1000-3000ms：更省资源，适合不需要即时响应的场景

### 白名单管理

白名单中的应用不会被自动静音，即使处于后台也保持正常音量。

**添加方式**：

1. **手动输入**：在"进程名"输入框中输入进程名称（如 `chrome.exe`），点击"添加"
2. **从运行中添加**：点击"刷新列表"获取当前运行的进程，从下拉框选择后点击"添加选中"

**移除方式**：

1. 在列表中选中一个或多个项目，点击"移除选中"
2. 点击"清空白名单"移除所有条目

**默认白名单**：

以下系统进程默认在白名单中：

- `explorer.exe` - Windows 资源管理器
- `SearchHost.exe` - Windows 搜索
- `RuntimeBroker.exe` - 运行时代理
- `ShellExperienceHost.exe` - Shell 体验主机
- `ApplicationFrameHost.exe` - 应用框架主机

### 导入/导出

- **导出配置**：将当前白名单和设置导出为 JSON 文件
- **导入配置**：从 JSON 文件导入配置（原配置自动备份为 `.backup` 文件）

## 工作原理

### 核心流程

```
┌─────────────────────────────────────────────┐
│              监控循环 (每 N 毫秒)              │
├─────────────────────────────────────────────┤
│                                             │
│  1. 获取当前前台窗口进程 PID                   │
│              ↓                              │
│  2. 枚举所有音频会话进程                      │
│              ↓                              │
│  3. 对每个音频进程判断：                       │
│     ├─ 是前台进程？ → 取消静音（如被我们静音）  │
│     ├─ 在白名单中？ → 取消静音（如被我们静音）  │
│     └─ 后台+非白名单？ → 执行静音              │
│              ↓                              │
│  4. 清理已退出的进程记录                       │
│                                             │
└─────────────────────────────────────────────┘
```

### 音量恢复机制

程序使用 `_previous_states` 字典追踪所有由本程序静音的进程。当检测到以下情况时自动恢复音量：

1. **应用切换到前台**：进程从后台变为前台时，立即取消静音
2. **应用加入白名单**：白名单更新后，已在白名单中的被静音应用自动恢复
3. **功能被禁用**：关闭自动静音功能时，所有被静音的应用全部恢复
4. **程序退出**：退出时自动恢复所有被静音应用的音量

### 前后台检测

通过 Windows API `GetForegroundWindow()` 获取当前活跃窗口句柄，再通过 `GetWindowThreadProcessId()` 获取对应的进程 ID，实现精确的前后台状态识别。

## 配置文件

配置文件位于 `config/config.json`，格式如下：

```json
{
  "whitelist": [
    "explorer.exe",
    "chrome.exe",
    "spotify.exe"
  ],
  "settings": {
    "check_interval_ms": 500,
    "auto_start": false,
    "muted_volume": 0,
    "restore_volume": 100,
    "enabled": true
  }
}
```

### 配置项说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `whitelist` | string[] | 白名单进程名列表（不区分大小写） |
| `settings.check_interval_ms` | int | 检测间隔（毫秒），范围 200-3000 |
| `settings.auto_start` | bool | 是否开机自启动 |
| `settings.muted_volume` | int | 静音时音量（保留字段，当前使用静音方式） |
| `settings.restore_volume` | int | 恢复时音量（保留字段，当前恢复原始状态） |
| `settings.enabled` | bool | 是否启用自动静音功能 |

## 日志系统

### 日志位置

日志文件保存在 `logs/volume_controller.log`。

### 日志轮转

- 单个日志文件最大 5MB
- 保留 3 个历史备份
- 自动轮转，无需手动清理

### 日志级别

| 级别 | 说明 |
|------|------|
| DEBUG | 详细的静音/取消静音操作记录 |
| INFO | 前台进程切换、静音/恢复操作、配置变更 |
| ERROR | 操作失败、异常情况 |
| CRITICAL | 程序致命错误 |

## 开机自启动

### 通过设置界面启用

1. 右键托盘图标 → 设置
2. 勾选"开机自动启动"
3. 程序将自动写入注册表

### 注册表位置

```
HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run
键名: VolumeController
键值: "Python路径\python.exe" "项目路径\main.py"
```

### 取消自启动

在设置界面取消勾选"开机自动启动"，或手动删除注册表中对应键值。

## 常见问题

### Q: 为什么某些应用切换到前台后仍然没有声音？

A: 可能的原因：
1. 该应用被手动静音（非本程序操作），本程序只会恢复由自己静音的应用
2. 应用切换过快，检测间隔尚未到达。可尝试缩短检测间隔
3. 应用使用了独立的音频引擎，不受系统音量会话控制

### Q: 如何让某个应用始终有声音？

A: 将该应用的进程名添加到白名单中。可以在设置界面的"白名单管理"标签页操作，也可以直接编辑 `config/config.json` 文件。

### Q: 如何找到应用的进程名？

A: 以下几种方式：
1. 打开任务管理器 → 详细信息标签，查看进程名称
2. 在设置界面的白名单管理中点击"刷新列表"，从下拉框中选择
3. 使用命令行：`tasklist | findstr 应用名`

### Q: 程序退出后声音没有恢复怎么办？

A: 正常情况下程序退出时会自动恢复所有被静音的应用。如果异常退出导致未恢复，可以：
1. 重新启动程序，然后通过托盘菜单正常退出
2. 手动在系统音量混合器中取消静音

### Q: 程序占用资源过高怎么办？

A: 尝试以下调整：
1. 增大检测间隔（如从 500ms 调整为 1000ms）
2. 检查白名单是否过长
3. 查看日志中是否有频繁的异常记录

### Q: 支持哪些类型的音频应用？

A: 支持所有使用 Windows 音频会话 API 的应用，包括：
- 大部分桌面应用程序
- UWP 应用
- 浏览器
- 游戏客户端

不支持使用独占模式音频或绕过 Windows 音频引擎的应用。

## 项目结构

```
VolumeController/
├── main.py              # 主程序入口，整合所有模块
├── volume_controller.py # 音量控制模块，管理音频会话的静音/恢复
├── process_monitor.py   # 进程监控模块，检测前后台状态
├── config_manager.py    # 配置管理模块，白名单/设置/导入导出
├── gui.py               # 图形界面模块，托盘图标和设置窗口
├── logger.py            # 日志模块，日志记录和轮转
├── requirements.txt     # Python 依赖列表
├── config/
│   └── config.json      # 配置文件（自动生成）
└── logs/
    └── volume_controller.log  # 日志文件（自动生成）
```
